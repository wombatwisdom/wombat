Reads messages from NATS subjects using wombatwisdom components

```yml
# Config fields, showing default values
input:
  label: ""
  ww_nats:
    url: nats://localhost:4222
    subject: "" # No default (required)
    queue: ""
    batch_count: 1
    name: wombat
    auth:
      jwt: ""
      seed: ""
```

Seamless integration of wombatwisdom NATS input component into Wombat.

This component uses the wombatwisdom NATS implementation under the hood while providing
a native Benthos interface. All wombatwisdom features are available including advanced
connection management, authentication, and monitoring capabilities.

## Connection Management

The component automatically handles connection lifecycle and provides robust error
handling and reconnection logic through the wombatwisdom framework.


## Fields

### `url`

URL of the NATS server to connect to. Multiple URLs can be specified by separating them with commas.


*Type*: `string`

*Default*: `"nats://localhost:4222"`

### `subject`

The subject to subscribe to. May contain wildcards.


*Type*: `string`


### `queue`

Optional queue group to join for load balancing.


*Type*: `string`

*Default*: `""`

### `batch_count`

Maximum number of messages to fetch at a time.


*Type*: `int`

*Default*: `1`

### `name`

Optional connection name for monitoring.


*Type*: `string`

*Default*: `"wombat"`

### `auth`

Authentication configuration


*Type*: `object`


### `auth.jwt`

User JWT token for authentication


*Type*: `string`

*Default*: `""`

### `auth.seed`

User seed for authentication


*Type*: `string`

*Default*: `""`
